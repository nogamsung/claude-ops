`memory/MEMORY.md` 에 기억을 누적·검색합니다.

**명령:** $ARGUMENTS

> 💡 **자동 컨텍스트 로드**: `CLAUDE.md` 규칙에 따라 Claude는 세션 시작 시 `memory/MEMORY.md` 를 자동으로 참조합니다. 별도 `/memory show` 호출은 불필요.

---

## `add <내용>` — 새 기억 추가

1. `memory/MEMORY.md` 를 읽습니다 (없으면 새로 생성)
2. 입력을 분석해 카테고리 판단:
   - **결정 (Decision)**: 기술 선택, 아키텍처 결정, 방향 변경
   - **규칙 (Rule)**: 코딩 컨벤션, 금지 패턴, 팀 규칙
   - **참고 (Reference)**: 외부 링크, 관련 파일 경로, API 문서
   - **교훈 (Lesson)**: 삽질 기록, 해결한 버그, 주의사항
3. 파일 끝에 추가:

```markdown
## YYYY-MM-DD: [한 줄 제목]

**카테고리:** 결정 | 규칙 | 참고 | 교훈

[내용 상세]

**관련 파일:** (있으면)
```

4. 추가한 내용을 보여주고 확인.

---

## `search <키워드>` — 검색

`memory/MEMORY.md` 에서 키워드가 포함된 섹션을 출력. 없으면: "관련 기억을 찾을 수 없습니다."

---

## 인수 없이 호출된 경우

짧은 도움말을 출력하고 종료합니다:

```
/memory 는 기억을 추가·검색하는 커맨드입니다.

  /memory add <내용>     — 새 결정·교훈 기록
  /memory search <키워드> — 기존 기억 검색

전체 내용은 세션 시작 시 자동 로드되므로 별도 조회는 불필요합니다.
직접 확인하려면: cat memory/MEMORY.md
```

---

## 자동 기록 트리거 (사용자 요청 없이 자동 추가)

아래 상황이 발생하면 **대화 흐름 중에 자동으로** `memory/MEMORY.md` 를 업데이트합니다. 별도 확인 없이 기록 후 "memory에 저장했습니다." 한 줄만 언급.

| 트리거 | 기록 내용 |
|--------|----------|
| `/plan` 승인 후 구현 시작 | 구현할 기능, 선택한 설계 방식, 이유 |
| `/rule` 로 새 규칙 추가 | 어떤 실수였는지, 추가된 규칙 요약 |
| 복잡한 버그 해결 | 원인, 해결 방법, 재발 방지 포인트 |
| 외부 라이브러리/API 도입 결정 | 선택 이유, 대안으로 고려했던 것 |
| 아키텍처 또는 폴더 구조 변경 | 변경 전/후, 변경 이유 |
| 성능 문제 발견 및 해결 | 병목 지점, 해결 방법, 측정 결과 |

---

## 사용 예시

```
/memory add JWT에서 세션으로 변경     # 새 기억 수동 추가
/memory search 인증                   # "인증" 키워드 검색
```

---

> **구분**
> - `memory/MEMORY.md` — 맥락·히스토리 (왜 이 결정을 했는가)
> - `CLAUDE.md` — 규칙 (앞으로 어떻게 해야 하는가)
